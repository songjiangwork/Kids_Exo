# 开发约定

## 工作方式

本项目优先采用测试驱动开发（TDD）：

1. 在实现新行为前，先写能够表达需求的测试。
2. 先运行测试，确认测试会因为缺少该行为而失败。
3. 编写满足需求的最小实现。
4. 运行全部测试，确认新行为通过且没有破坏既有行为。
5. 只在测试保护下进行后续重构。

## 第一版的测试重点

- 练习卷 preset 能够表达 A4 PDF 输出、英文 locale、区域格式和每个区域自己的题型插件选项。
- 分配律插件能够生成按数位加法拆分与接近整十/整百减法拆分的题。
- 相同随机种子能够复现相同题目，便于以后复印或追溯练习卷。
- 命名书写格式能产生热身区和专项区所需的英文纸面内容。
- PDF 输出遵循 A4 页面和英文标题约定。
- 新的 beginner preset 的专项训练区默认使用 `count = 30`。
- 特殊规则存在易漏边界时，beginner preset 应有测试保证边界题稳定出现，例如尾积需要写为 `09` 的乘法题。

## 实现原则

- 题目生成逻辑不依赖 PDF 渲染器。
- PDF 渲染器只消费结构化练习卷数据，不判断数学策略。
- 区域的教学标题、英文说明和示例属于练习卷展示数据，渲染器不硬编码题型文案。
- 题型特有设置由题型插件拥有；通用练习卷设置不吸收数学领域细节。
- 输出渲染器的专属设置（如 PDF 纸张大小）不进入题型或生成出的数学内容模型。
- 首版允许使用固定的命名格式，未来再扩展为安全的可配置模板语言。
- 首版优先使用 Python 标准库实现，减少运行和安装门槛；需要引入依赖时再明确记录原因。
